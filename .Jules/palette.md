## 2025-02-18 - Keyboard Accessibility and Focus Indicators
**Learning:** This app requires explicit handling for keyboard accessibility. A pattern of `outline-none focus:ring focus:ring-sky-600` mixed with `hover:bg-slate-800` is needed to provide visible focus indicators and hover states for interactive components.
**Action:** When working on interactive elements, always ensure keyboard accessibility by appending the consistent `outline-none focus:ring focus:ring-sky-600` Tailwind classes, alongside relevant hover state styling.
